# Java Standards

Java is used for backend services, microservices (Spring Boot), and Android development.

## Quick Reference

| Rule | Value | Source |
|------|-------|--------|
| Line length (column limit) | 100 characters | Google Java Style Guide |
| Indentation | 2 spaces (NOT 4) | Google Java Style Guide |
| Brace style | K&R / Egyptian | Google Java Style Guide |
| Class names | `PascalCase` | Oracle + Google |
| Method/variable names | `camelCase` | Oracle + Google |
| Constants | `UPPER_SNAKE_CASE` | Oracle + Google |
| Max method length | 40 lines | Team convention |
| Max class length | 500 lines | Team convention |
| Min Java version | Java 17 (LTS) | Team convention |
| Min test coverage | 80% business logic | Team convention |

> **Note:** Google Java Style Guide uses **2-space** indentation, which differs from Oracle's historical 4-space recommendation. We follow Google's guide for consistency with our other languages.
